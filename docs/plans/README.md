# Archived Planning Notes

This directory contains historical implementation plans, review notes, and roadmap drafts.

These files are not the active backlog and should not be read as current product commitments. They are kept for context on prior design decisions, shipped milestones, and review findings.

Current work should be tracked through GitHub issues, pull requests, and release notes. If a historical plan is revived, create a fresh issue or implementation plan that reflects the current codebase and product positioning.

## Reading Guide

- Treat dates in filenames as the plan creation date, not the current status.
- Verify claims against `README.md`, `CHANGELOG.md`, source code, and tests before using them in public docs.
- Prefer small, focused follow-up issues over reviving an old multi-phase plan wholesale.
